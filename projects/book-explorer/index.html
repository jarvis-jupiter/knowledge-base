Project Structure

Monorepo Architecture

pnpm Workspaces + Turborepo
Shared Packages

@book-explorer/domain

Pure TypeScript types & schemas shared between API and Web. Book, Bookmark, User types, Zod schemas, Result<T> monad.

@book-explorer/db

Prisma client, schema, and migrations. Single source of truth for the database model. Generated client consumed by the API.

Turborepo caching

Tasks only re-run on changed packages. build, typecheck, test outputs are cached — skipped when inputs unchanged.

Design Pattern

Hexagonal Architecture (Ports & Adapters)

Layer Responsibilities
1

Domain Core

Pure business logic — use cases (AddBookmarkUseCase, SearchBooksUseCase). No framework dependencies. Fully testable in isolation.

2

Ports (Interfaces)

TypeScript interfaces the domain depends on: BookSearchPort, BookmarkRepositoryPort, UserRepositoryPort.

3

Adapters (Implementations)

Concrete implementations wired at startup: Google Books HTTP adapter, Prisma repository adapters, Express HTTP router, Clerk auth middleware.

4

Dependency Inversion

Domain never imports adapters. Adapters are injected at composition root (index.ts). Swapping Prisma for another DB requires only a new adapter.

Design Patterns

Functional Programming Patterns

🎯

Result<T> Monad

Errors are part of the return type — never thrown. The compiler forces callers to handle both success and failure paths.

type Result<T> =

| { ok: true; value: T }

| { ok: false; error: AppError }

🔧

No Classes

All code written as pure functions and factory functions. Use cases are objects with an execute method created via factory functions, not class instantiation.

const createAddBookmark =

(repo: BookmarkRepo) => ({

execute: async (input) => ...

})

✅

Test-Driven Development

Every feature written test-first. Unit tests for domain logic, integration tests for adapters, Playwright E2E for user flows. CI enforces all checks before merge.

61 Unit Tests 60 E2E Tests Page Object Model
🛡️

Zod Validation

All external inputs — HTTP request bodies, query params, API responses — validated at the boundary with Zod schemas. Invalid data never reaches domain logic.

🏷️

Discriminated Unions

Errors carry a kind discriminant: ValidationError, NotFound, Conflict, Forbidden. Routes map each kind to the correct HTTP status code.

🔒

Readonly by Default

All domain types use readonly properties. TypeScript strict mode enforced. Biome linter ensures consistent code style and catches common errors at CI time.

Data Layer

Database Schema & Prisma

Key Design Decisions
💾

Denormalised bookmark data

Book metadata (title, cover, authors) stored in bookmark row — avoids a join to Google Books API on every bookmark fetch. Books aren't stored in our DB.

🔑

Clerk → DB user sync

Clerk handles auth. DB users table syncs via webhooks (user.created/updated/deleted) and auth middleware upsert for race condition resilience.

🚫

Unique constraint prevents duplicates

@@unique([userId, bookId]) at DB level, with P2002 error code caught in the repository and mapped to a Conflict domain error.

⚡

Indexes on hot queries

@@index([userId]) for bookmark list; @@index([createdAt]) for sorted queries. clerkId covered by its unique constraint.

Frontend

Remix Architecture

Key Remix Principles Applied
🌐

Server-side rendering by default

Loaders run on the server — no client-side data fetching waterfalls. Search results, bookmarks, and book details are SSR'd for instant first paint and SEO.

📝

Progressive enhancement

<Form method="get"> for search works without JS. useFetcher enhances bookmark button to post without page navigation when JS is available.

⚡

HTTP caching on search

Cache-Control: public, max-age=300, stale-while-revalidate=600 — search results cached 5 minutes in browser and CDN, served stale for up to 10 minutes while revalidating.

🔐

Clerk integration

rootAuthLoader in root.tsx. Per-route auth via getAuth() in loaders. Unauthenticated bookmark attempts redirect to /sign-in?redirect_url=....

Infrastructure

Docker Multi-Stage Build

Security & Reliability
🔒

Non-root user

Container runs as nodeuser — not root. Industry-standard security practice to limit blast radius of any container escape.

❤️

HEALTHCHECK

Docker checks /health every 30s. Railway uses the container health status to determine when a deployment is ready, and restarts crashed containers automatically.

🛑

Graceful shutdown

SIGTERM handler calls server.close() then prisma.$disconnect(). Existing requests finish before the process exits. 10s forced exit timeout.

📦

pnpm deploy isolation

pnpm deploy --prod creates a clean deployment dir with only production dependencies and a flat node_modules — no workspace symlinks in production.

Quality Assurance

Testing Strategy

E2E: Page Object Model
📄

Page Objects

HomePage, SearchPage, BookDetailPage encapsulate locator logic. Tests read like user stories, not CSS selector queries.

🔧

Fixtures

Playwright fixtures inject page objects into tests. test.extend() provides typed searchPage, bookDetailPage per test.

🐳

Docker Compose in CI

E2E tests run against a real Docker Compose stack (postgres + api + web) — same images deployed to production. Catches Docker-specific bugs before they hit Railway.

🔄

2 retries on CI

Flaky network-dependent tests (Google Books API latency) get 2 automatic retries on CI. Stable tests pass first attempt.
